# Meta（Facebook）出稿 — 基本フロー

## この章で学ぶこと

Meta広告の出稿を、依頼を受けてから公開するまでの一連の流れで理解する。
出稿の種類（キャンペーン予算・セット予算・tcpa・ASC等）に関わらず、共通する基本フローをまず身につける。

---

## 出稿の全体像

```
CR数と設計数確認 → CRアップロード → Beyond記事作成 → キャンペーン作成 → 広告セット設定 → 広告設定 → 確認 → 公開
```

### なぜこの順番なのか

- CRアップロードは時間がかかる → **先に開始して、待ち時間でBeyond記事を作る**のが効率的
- キャンペーン → 広告セット → 広告は**階層構造**になっている。上から順に作らないと下の階層が作れない
- 確認を最後にまとめてやることで、設定ミスを一括で検知できる

---

## Step 0: 依頼の確認

### やること

- 出稿依頼スプシで自分の名前が入っている依頼を確認する
- 出稿依頼スプシ: [リンク](https://docs.google.com/spreadsheets/d/1ehFQ5xv5Ymvb4cAlOlS8b8AFcIJhTuPL61RlJZE8RSY/edit?gid=1922688053#gid=1922688053)

### 確認する情報

| 項目 | スプシの列 | 確認ポイント |
|------|-----------|-------------|
| 案件名 | - | どのクライアント・メニューか |
| 媒体 | - | Meta / TikTok / X 等 |
| CR数 | - | 何本のクリエイティブを出すか |
| 設計パターン | - | 冒頭 / 年齢別 / 地域別 等 |
| キャンペーン名 | F列 | 命名規則に従っているか |
| 予算設定 | N列・P列 | キャンペーン予算 or セット予算、金額 |
| ピクセル・CV設定 | U列 | データセット・コンバージョンイベント・アトリビューション |

### なぜ確認が大事か

> 配信設定が適切に行われないとCVが発生しない。
> 例：男性用の脱毛サロンの広告を女性に配信してしまったら、クリエイティブが無駄になる。
> **最初のうちは速さより正確さ。** 依頼内容通りにキャンペーン設定ができているかが最重要。

### 不明点があったら

- **そのままにしない。** なんとなくで作って配信内容がズレた場合、損失が生まれる
- 確認先: 依頼者（運用者）、アド管理メンバー、他のアルバイト
- 聞いたことで責められることは絶対にない。**聞かずにミスる方が問題**

## Step 1: CRアップロード

CRをGoogleドライブからダウンロードし、Meta広告マネージャーのメディアライブラリにアップロードする。

### 手順

1. 出稿したいクリエイティブをGoogleドライブからダウンロードする
2. ビジマネの左上「キャンペーン」に出稿したいアカウント名を検索する
3. 左の三本線メニュー → **メディアライブラリ**を開く
4. 「広告アカウントのメディア」→ **Select Ad Account** → アカウント名を検索して選択
5. **動画** → **Add Media** → **Add video** → ダウンロードしたCRをアップロード
6. 緑になって「OK」が出ればアップロード完了

### ビジマネとアカウントの関係

```
Business01
Business03
Business08    ← ビジマネの中にアカウントが存在する
Business11
Business13
Business14
```

- アカウントがどのビジマネに属するかは、**アカウント名の先頭の数字**で判断する
- 例: `3-001_TCB-クマ再生注射女性-静止画` → **Business03**

### 注意点

- FBは基本的に**動画のみ**
- アップロードには時間がかかる → **待ち時間でStep 2のBeyond記事作成を進める**

---

## Step 2: Beyond記事作成

記事ページを複製し、アフィリンクを設定する。

### なぜアフィリンクを分けるのか

> キャンペーンごとに遷移先のURLを分けている。中身は同じだがURLが違う。
> **理由: 効果測定を正確にするため。**
> 2種類のキャンペーンで最終的な遷移先が同じだと、CVが発生してもどちらのキャンペーンの成果か測定できなくなる。
> どのクリエイティブでCVがついたかは非常に重要な情報。それを識別するためにキャンペーンごとにアフィリンクを分ける。

### 手順

1. Beyondにログインする
2. 該当案件のフォルダを開く
3. 複製元の記事を複製 → キャンペーン名に修正する
4. **version**を開く（別タブで開いた方が楽）
5. 「リンク置換」マークを開く → 全て選択 → スプシのアフィリンクを入力 → **置換**
6. **更新ボタン**を押す（オレンジ → 青になればOK）
7. 元のタブに戻って「記事URL」をコピーする
8. スプシに貼り付ける
9. ドライブURL抽出にドライブURLを貼って実行する

### ポップアップ記事の場合

ポップアップ記事の指定がある場合は、アフィリンク置換後に追加作業が必要:

1. アフィリンク置換・更新 → 左の「ポップアップ」を選択
2. 該当のversionを選択 → リンク＋ → 出てきたリンクを選択
3. 本番反映を押して完了 → 戻って記事URLをコピーする

---

## Step 3: キャンペーン作成

### 新規作成の場合

1. キャンペーンを作成する
2. キャンペーンの目的を選択 → **手動作成の売上キャンペーン**
3. 設計数に合わせて広告セット・広告数を作成する
   - 3つの点 → 複製 → 既存のキャンペーン → 複製したい数 → 複製する
4. 広告セット階層の**オーディエンス**を切り替える
   - Advantageオーディエンス → **元のオーディエンスオプション**に切り替える → 「元のオーディエンスを使用」を選択

### 複製の場合（基本はこちらが多い）

1. キャンペーン名にあるキーワードでキャンペーンを絞る（運用者名、年齢など）
2. キャンペーンの中身を確認して、似たキャンペーンを複製する
3. 「現在の設定を使って複製」を選択
4. 複製数は **1**
5. キャンペーン名をスプシのF列の内容に修正する
6. キャンペーン詳細: **売上**
7. スプシN列が「キャンペーン予算」なら、キャンペーン予算を**ON**にしてP列の金額を記入する
8. 設計数に合わせて広告セット・広告数を増やす or 削除する

---

## Step 4: 広告セット設定

### 広告セット名の修正

1. 広告セットを修正する（一気にやった方が速い）
2. 修正したい広告セットを全て選択 → 編集横の▼ → **名前**
3. 出稿内容によって広告セット名を変える:
   - **冒頭**: 広告セット名 → CR名
   - **年齢別**: 広告セット名 → 年齢
   - **地域別**: 広告セット名 → 駅名（院＋駅名）

### 広告セットの内容修正

4. 全て選択 → **編集**
5. **ウェブサイト** → データセット・コンバージョンイベント・アトリビューション設定はスプシU列の内容を選択
6. **開始日**は翌日の00:00
7. **地域・年齢・性別**を設定する
   - 年齢別の場合: 広告セット名に合わせて年齢を修正 → 下書きとして保存
   - 地域別の場合: 1キャンペーンずつ地域（駅名）を選択していく
     - 駅名検索 → 駅の場所が合っていそうなら選択（違う駅が出ることもあるので**要注意**）→ 範囲選択
     - 駅名で出てこなければ出稿依頼スプシの「店舗一覧」シートから住所検索する

### 注意点

> セールスキャンペーンが使用できるアカウントでキャンペーン予算出稿する場合は、**「提案として使用する」のチェックを外す**。
> チェックを外さないと、意図していないターゲティングに配信してしまう。

---

## Step 5: 広告設定

### 広告名の修正

1. 広告名を広告セット名に合わせて修正する
   - 例: 広告セット名が「52」なら、広告名も「52」に修正

### ページ選択

2. ページを選択する
   - 出稿依頼シートに記載されているページ名、またはDBシートの「FBページ」シートで該当案件のページを検索

### URL設定

3. 広告のURLを修正する: 全て選択 → 編集横の▼ → **ウェブサイトURL**
4. Beyondで作成した記事URLを貼る → 下書きとして保存

### リファラ設定

5. **URLパラメーターを作成**する
6. キャンペーンコンテンツの `{{ad.name}}` を選択する

> リファラ = ユーザーがサイトにアクセスする際に利用したリンク元のページ情報。
> これにより、どの広告経由でユーザーが来たかを追跡できる。

### CR設定

7. 広告を全選択 → 編集横の▼ → **メディア**
8. 修正したい広告を選択 → 広告名をコピー → 検索して該当CRを選択
   - 年齢別: 該当の年齢CR名をコピー
   - 地域別: 該当の院・駅名のCR名をコピー
9. 出てきたCRを選択 → 次へ
   - CRが重複した場合は**一番左の新しいCR**を選択
   - **グルグル（検索中）の状態だとエラーになる** → 動画が出てきた後に選択する
10. CR全て修正したら**下書きとして保存**

---

## Step 6: 確認

広告まで作成できたら、依頼内容と一致しているか全項目を確認する。
キャンペーン → 該当のCPNから編集 → 内容が問題ないか確認していく。

### チェック項目

- [ ] キャンペーン名
- [ ] 予算金額
- [ ] データセット・コンバージョン・アトリビューション設定
- [ ] 開始日
- [ ] 地域
- [ ] 年齢
- [ ] 性別
- [ ] Facebookページ
- [ ] 記事URL（※URLを取り違えると正しくユーザーの計測ができない）
- [ ] 動画（正しいCRが入っているか）
- [ ] CPN・広告セット・広告が全て選択されているか
- [ ] 配信ONになっているか

> 詳細な確認方法は [[03_Meta出稿_チェックリスト]] を参照。

---

## Step 7: 公開

1. 内容が問題なければ**公開**する
2. 配信オフになっていたら配信を**ON**にする

### 注意点

- **CR待ちの場合**: CR以外を作成・公開してCRを待つ（公開にしないと他の人が触れなくなるため）
- **右上の「確認して公開」が残っていないか要注意** → 残っていたら正しく配信されていないので該当CPN確認する

---

## 公開後にやること

1. 出稿依頼スプシにチェックを入れる
2. 運用者へ報告する
