# 広告出稿マニュアル — チャットボット用ナレッジベース

> このドキュメントはチャットボットのコンテキスト/知識ベースとして使用する。
> 新人運用者が出稿業務を覚えるためのマニュアル全内容を構造化してまとめている。

---

## 1. 出稿の基本概念

### 出稿とは
Meta、TikTok、Xなどの媒体で広告の配信設計を行うこと。「どんな広告を流したいか」「どんな人に配信したいか（年齢、地域、性別など）」を設定して、広告を世に出す作業。

### なぜ出稿が重要か
- 運用者は「制作 → 分析」のサイクルを回すことが本業
- 出稿を巻き取ることで、運用者がより制作に注力でき、売上が上がる
- 出稿の正確さが計測の正確さに直結する → ミスると成果が測れなくなる

### 売上が生まれるまでの流れ
ユーザー（SNSを見ている人）→ クリエイティブ（広告）が流れてくる → クリックすると記事に飛ぶ（自社で作成）→ 記事を見て次のボタンをクリック → LP（クライアントの予約フォーム）→ 予約完了 → サンクスページ到達 = コンバージョン（CV）発生

- 1CV = 単価分の売上（例: 単価2万円なら1CV = 2万円）
- 利益 = 単価 - 消化金額（広告に使った費用）
- 消化金額を抑えてCVを取るほど利益が大きくなる

### CV計測の2つの仕組み
出稿した広告の成果（CV）がどう計測されるか。現在、2つの方式が混在している。

#### 旧方式（ASPリンク）
- ASPが発行するアフィリンクを使う
- キャンペーンごとにリンクを分ける（1CP = 1リンク）
- CP名の一部で成果を突き合わせる
- リンクを分けないとどのCPのCVか特定できない

#### 新方式（CATS）
- 第三者計測ツール「CATS」を使う
- 媒体が発行するCPID/ADIDでCVを紐づけ
- 同じ案件なら1つの記事URLで全CP運用可能
- リンク管理の工数が大幅に減る

#### 見分け方: キャンペーン名の「AXAD」
- 旧: `TM2-1_白井_エミナル_A地域直LP_TikTok_AX049_0117_記事013_...` → **AX049 = ASP番号 → 旧方式**
- 新: `TM1-2_加藤_TCB-脂肪溶解_通常LP_FB_AXAD01_0112_記事002_...` → **AXAD = 新方式（CATS）**

#### 覚え方
- キャンペーン名に「AXAD」が入っていれば → CATS（新方式）
- 入っていなければ → 旧方式（ASPリンク）
- 全てのリンクがCATS連携できるわけではない（ASP側の対応状況による）ので、当面は新旧が混在する

### キャンペーン・広告セット・広告の3階層構造
```
キャンペーン（箱）   → 名前、予算を設定
└── 広告セット（ターゲティング） → 年齢、性別、地域、開始日を設定
    └── 広告（配信物）     → CR（動画）、記事URL、ページを設定
```
この3つが全て設定されて初めて広告が配信される。基本は翌日0時から配信開始。

### キャンペーン構造の型
表記: キャンペーン数 - 広告セット数 - 広告数。依頼時に「1-1-Nで」のように指示されるので、この読み方を覚える。

| 型 | 意味 | 用途 |
|----|------|------|
| 1-1-1 | 最小構成 | 単体テスト |
| 1-1-N | 同じターゲットでCR比較 | 複数CRのA/Bテスト |
| 1-N-1 | 同じCRでターゲット比較 | 年齢別・地域別の検証 |
| 1-N-N | ターゲット×CRの組み合わせ | 本格的な検証 |

#### 1-1-N の例（同じターゲットで複数CR比較）
```
キャンペーン
└── 広告セット（25-34歳・女性・関東）
    ├── 広告A
    ├── 広告B
    └── 広告C
```

### 心構え
- 不明点は他の人に遠慮なく聞く
- 不明点をそのままにしない — なんとなくで作ると損失が生まれる
- 最初のうちは速さより正確さ — 慣れたら勝手に早くなる

## 2. Meta（Facebook）出稿

### 2-1. 基本フロー（全体像）
```
CR数と設計数確認 → CRアップロード → Beyond記事作成 → キャンペーン作成 → 広告セット設定 → 広告設定 → 確認 → 公開
```
CRアップロードは時間がかかるため、先に開始して待ち時間でBeyond記事を作るのが効率的。

### Step 0: 依頼の確認
出稿依頼スプシで自分の名前が入っている依頼を確認する。

確認する情報:
- 案件名: どのクライアント・メニューか
- 媒体: Meta / TikTok / X 等
- CR数: 何本のクリエイティブを出すか
- 設計パターン: 冒頭 / 年齢別 / 地域別 等
- キャンペーン名: 命名規則に従っているか
- 予算設定: キャンペーン予算 or セット予算、金額
- ピクセル・CV設定: データセット・コンバージョンイベント・アトリビューション

### Step 1: CRアップロード
1. スプシのCRフォルダURLからGoogleドライブを開く
2. 動画ファイルを全選択してダウンロード（ZIP）→ 解凍
3. Meta広告マネージャー → すべてのツール → メディアライブラリ
4. 広告アカウントをスプシの「広告アカウント名」で検索して選択
5. 動画タブ → Add Media → Add Video → ファイル選択 → アップロード
6. 緑バーで完了 → OK

ビジマネとアカウントの関係:
- Business01、Business03、Business08、Business11、Business13、Business14 の中にアカウントが存在
- アカウント名の先頭の数字で所属ビジマネを判断（例: 3-294 → Business03）

### Step 2: Beyond記事作成
Squad beyond上で既存の記事を複製し、新しいキャンペーン用の記事URLを作成する。

なぜアフィリンクを分けるのか: キャンペーンごとに遷移先URLを分けることで、どのキャンペーンの成果か正確に測定するため。

手順:
1. スプシの「複製元beyondページ名」を確認
2. Squad beyondで該当フォルダ → 複製元記事を検索
3. 三点メニュー → beyondページ複製 → スプシの「beyondコピペ用CPN名」を貼り付け → 複製
4. 複製した記事を検索 → タイトルクリック → 編集画面
5. version → リンク置換 → 全て選択 → スプシのアフィリンクを入力 → 置換
6. 更新ボタン（オレンジ → 青になればOK）
7. 記事URLをコピー → スプシに貼り付け

ポップアップ記事の場合: アフィリンク置換後に追加作業が必要
- 左の「ポップアップ」選択 → 該当version → リンク＋ → リンク選択 → 本番反映

### Step 3: キャンペーン作成

新規作成の場合:
1. +作成 → 手動作成の売上キャンペーン
2. 設計数に合わせて広告セット・広告数を複製
3. 広告セット階層でAdvantageオーディエンス → 元のオーディエンスに切り替え

複製の場合（基本はこちらが多い）:
1. キャンペーン名のキーワードで絞る
2. 似たキャンペーンを複製 → 現在の設定を使って複製 → 複製数1
3. キャンペーン名をスプシF列の内容に修正
4. キャンペーン詳細: 売上
5. スプシN列が「キャンペーン予算」ならONにしてP列の金額を記入
6. 設計数に合わせて広告セット・広告数を調整

### Step 4: 広告セット設定

広告セット名の修正:
- 冒頭: 広告セット名 → CR名
- 年齢別: 広告セット名 → 年齢
- 地域別: 広告セット名 → 駅名（院＋駅名）

広告セットの内容修正:
1. 全て選択 → 編集
2. ウェブサイト → データセット・コンバージョンイベント・アトリビューション設定（スプシU列）
3. 開始日: 翌日の00:00
4. 地域・年齢・性別を設定
   - 地域別: 駅名検索 → 範囲選択（出てこなければ「店舗一覧」シートから住所検索）


### Step 5: 広告設定

1. 広告名の修正: 広告セット名に合わせて修正（全選択 → 編集▼ → 名前）
2. ページ選択: スプシ記載のページ名またはDBシートの「FBページ」で検索
3. URL設定: 全選択 → 編集▼ → ウェブサイトURL → Beyond記事URLを貼る
4. リファラ設定: URLパラメーターを作成 → キャンペーンコンテンツで {{ad.name}} を選択
5. CR設定: 全選択 → 編集▼ → メディア → 広告名で検索して該当CRを選択

リファラとは: ユーザーがサイトにアクセスする際に利用したリンク元のページ情報。どの広告経由でユーザーが来たかを追跡できる。

注意: CRの検索でグルグル（読み込み中）の状態だとエラーになる。動画が表示された後に選択すること。

### Step 6: 確認
確認項目: キャンペーン名、予算金額、データセット・コンバージョン・アトリビューション、開始日、地域、年齢、性別、Facebookページ、記事URL、動画、全選択されているか、配信ON

### Step 7: 公開
1. 右上の「確認して公開」→「公開する」
2. 配信オフなら配信ON
3. スプシにチェック → 運用者へ報告

注意:
- CR待ちの場合: CR以外を作成・公開してCRを待つ（公開にしないと他の人が触れなくなる）
- 「確認して公開」が残っていないか確認 → 残っていたら正しく配信されていない

### 2-2. 種類別差分

| 種類 | 予算の場所 | 特徴 |
|------|-----------|------|
| キャンペーン予算（冒頭） | キャンペーン階層 | 基本形。同じ設定で複数CRを検証 |
| キャンペーン予算（年齢別） | キャンペーン階層 | 年齢セグメントごとに広告セットを分ける |
| キャンペーン予算（地域別） | キャンペーン階層 | 地域（駅名）ごとに広告セットを分ける |
| セット予算 | 広告セット階層 | 広告セットごとに予算を設定 |
| tcpa | キャンペーン階層 | 単価の目標を設定して最適化 |

#### キャンペーン予算（基本形）
基本フローそのまま。キャンペーン予算ON、予算金額はスプシP列。

#### セット予算
- キャンペーン階層: キャンペーン予算OFF
- 広告セット階層: 「1日の予算」に金額を入れる
- 用途: 広告セットごとに予算を個別管理したい場合

#### tcpa（Target Cost Per Action）
- キャンペーン階層: キャンペーン予算ON、入札戦略を「単価の目標」に変更
- 広告セット階層: 「結果の単価目標」に金額を入れる
- 用途: 1件あたりの獲得コスト（CPA）を指定金額に近づけるよう最適化

## 3. TikTok出稿

### 3-1. 基本フロー
```
CR数と設計数確認 → CRアップロード → Beyond記事作成 → キャンペーン作成 → 広告セット設定 → 広告設定 → 確認 → 公開
```

Metaとの主な違い:
- CRアップロード先が「クリエイティブライブラリ」（Metaは「メディアライブラリ」）
- 広告セットで「プレースメント設定」が必要
- 広告に「テキスト設定」が必要
- バルク出稿（Excel経由）という方法もある

### Step 0: 依頼の確認
Meta出稿と同じ。

### Step 1: CRアップロード
1. TikTok広告マネージャーでアカウント名を検索 → 選択
2. ツール → クリエイティブライブラリを選択
3. アップロード → CRをドロップまたはクリックしてアップロード
4. 緑のチェックマークがついたら完了

### Step 2: Beyond記事作成
Meta出稿と同じ手順。複製元ページ名とアフィリンクがTikTok用の値になる点だけ異なる。

### Step 3: キャンペーン作成
1. 広告 → ＋作成
2. 売上 → Webサイト → 手動キャンペーンを選択
3. 指定のキャンペーン名を入れる
4. キャンペーン予算最適化をON → 予算を入れる

### Step 4: 広告セット設定
1. 広告セット名を入れる
2. 手動プレースメントを選択 → **TikTokのみON**にする（Pangle・グローバルアプリバンドルはOFF）
3. ユーザーによるコメント・動画ダウンロード・動画シェアを許可を**OFF**にする
4. ロケーション・年齢・性別・言語を選択する
5. 開始日を指定する（翌日0時開始）

⚠️ 重要: プレースメントをTikTokのみにしないと、Pangleなど別の配信面にも流れてしまう。必ず確認すること。

### Step 5: 広告設定
1. 動画を追加 → クリエイティブライブラリで動画名を検索して選択
2. テキストを設定（指定があればそれを使用、なければ「ここからのお申込限定」）
3. URLを入れる（Beyondで作成した記事URL）
4. 誘導アクション: 標準 → 指定のアクション（例: 「詳細を見る」）のみチェック
6. 内容確認 → 公開

### Step 6: 確認 → 公開
1. 設定内容を全て確認
2. 問題なければ「すべて公開」
3. スプシにチェック → 運用者へ報告

### 3-2. 種類別差分

| 種類 | 予算の場所 | 特徴 |
|------|-----------|------|
| キャンペーン予算 | キャンペーン階層 | 基本形 |
| セット予算（最大化） | 広告セット階層 | セットごとに予算配分 |
| セット予算（tcpa） | 広告セット階層 | 単価目標を指定 |
| Smart+ | - | 管理画面で簡易設定 |

#### キャンペーン予算（基本形）
基本フローそのまま。差分なし。

#### セット予算（最大化）
- キャンペーン階層: キャンペーン予算を選択、設定部分は何も選択しない
- 広告セット階層: 「日予算」に金額を入れる

#### セット予算（tcpa）
- キャンペーン階層: キャンペーン予算を選択、設定部分は何も選択しない
- 広告セット階層: 「日予算」に金額を入れる + 入札戦略で単価目標を選択し目標金額を入れる

#### Smart+
TikTok側のAIがターゲティング・入札を自動最適化する簡易設定タイプ。
1. ＋作成 → Smart+キャンペーンを選択
2. 売上 → Webサイト
3. キャンペーン名・予算を入力
4. ターゲティング（性別・年齢・地域）を設定
5. プレースメント: TikTokのみON
6. CR・テキスト・URLを設定 → 公開

## 4. 用語集

| 用語 | 説明 |
|------|------|
| CR（クリエイティブ） | 広告として配信する動画素材 |
| CV（コンバージョン） | ユーザーが広告経由で予約完了など目標アクションを達成すること |
| CPA（Cost Per Action） | 1コンバージョンあたりの獲得コスト |
| CPN | キャンペーンの略称 |
| スプシ | スプレッドシート（出稿依頼シート）の略 |
| Beyond / Squad beyond | 記事作成・管理ツール。広告クリック後のLP前の記事ページを作成する |
| アフィリンク | アフィリエイトリンク。効果測定用にキャンペーンごとに分けたURL |
| ビジマネ | ビジネスマネージャー（Meta）。複数の広告アカウントを管理する上位組織 |
| プレースメント | 広告が配信される場所（TikTok、Pangle、グローバルアプリバンドル等） |
| アトリビューション | 広告接触からコンバージョンまでの帰属設定 |
| データセット / ピクセル | ウェブサイト上のユーザー行動を計測するためのコード |
| tcpa | Target Cost Per Action。目標CPA設定での入札戦略 |
| Smart+ | TikTokのAI自動最適化キャンペーン |
| リファラ | ユーザーがサイトにアクセスする際のリンク元情報 |
| UTMパラメーター | URLに付与するトラッキング用パラメーター（utm_source, utm_medium等） |

## 5. よくある質問（FAQ）

### Q: MetaとTikTokの一番大きな違いは？
A: CRのアップロード先（メディアライブラリ vs クリエイティブライブラリ）、TikTokではプレースメント設定（TikTokのみON）とテキスト設定が追加で必要。

### Q: プレースメントの設定を忘れるとどうなる？
A: TikTok以外の配信面（Pangle等）にも広告が流れてしまい、意図しない場所に予算が消化される。

### Q: 開始日はいつにすればいい？
A: 基本は翌日の00:00。

### Q: キャンペーン予算とセット予算の違いは？
A: キャンペーン予算はMeta/TikTokのAIが自動で各広告セットに予算配分する。セット予算は各広告セットに個別に予算を設定し、均等に予算を使える。

### Q: tcpaとは何か？
A: Target Cost Per Action。1件あたりの獲得コスト（CPA）を指定金額に近づけるよう最適化する入札戦略。予算を使い切ることより、指定した単価での獲得を優先する。

### Q: Smart+キャンペーンとは？
A: TikTokのAIがターゲティング・入札を自動最適化するキャンペーン。設定項目が少なくシンプルだが、ターゲティングや予算の指定は正確に行う必要がある。

### Q: CR待ちの場合はどうする？
A: CR以外を全て作成・公開してCRを待つ。公開にしないと他の人が触れなくなるため。

### Q: アフィリンクをなぜ分ける必要がある？
A: キャンペーンごとに遷移先URLを分けることで、どのキャンペーン・どのCRでCVが発生したか正確に測定するため。同じURLだとどのキャンペーンの成果か識別できなくなる。

### Q: URLパラメーター（リファラ）の設定は何のため？
A: どの広告経由でユーザーが来たかを追跡するため。Meta広告ではキャンペーンコンテンツで {{ad.name}} を選択する。

### Q: ビジマネ（ビジネスマネージャー）の見分け方は？
A: アカウント名の先頭の数字で判断する。例: 3-294_出稿練習用アカウント → Business03。ビジマネにはBusiness01, 03, 08, 11, 13, 14 がある。
